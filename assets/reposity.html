CHESS GAME

Descrição:

Arquitetura Geral O jogo de xadrez é desenvolvido utilizando a engine Unity com linguagem de programação C#. A arquitetura do jogo segue os princípios da Programação Orientada a Objetos (POO), onde cada entidade do jogo é representada como um objeto com propriedades e comportamentos específicos.
Componentes Principais
Tabuleiro: Representa o espaço onde o jogo de xadrez acontece. Pode ser implementado como uma matriz 8x8 de células, onde cada célula pode conter uma peça de xadrez.
Peças de Xadrez: Cada peça (rei, rainha, torre, bispo, cavalo e peão) é representada por uma classe individual que herda de uma classe base Piece. Cada classe de peça possui métodos para movimentação válida, captura de peças adversárias e validação de regras específicas de movimento. Jogador: Representa os jogadores do jogo (branco e preto). Cada jogador pode realizar movimentos alternados durante a partida. Controlador de Jogo: Gerencia o estado atual do jogo (início, andamento, fim), controlando as interações entre os jogadores, validando movimentos e verificando condições de vitória ou empate.
Funcionalidades Implementadas
Movimentação de Peças: Cada peça de xadrez possui um conjunto específico de movimentos válidos, implementados através de métodos na classe da peça. O controlador do jogo valida se os movimentos são legais de acordo com as regras do xadrez.
Captura de Peças: Quando uma peça se move para uma posição ocupada por uma peça adversária, a peça adversária é capturada e removida do tabuleiro. Regras Especiais: Implementação de regras
especiais do xadrez, como roque, en passant e promoção de peões.
Interface Gráfica: Utilização de elementos visuais na Unity para representar o tabuleiro, peças de xadrez e interações do jogador. Cada peça pode ser associada a um modelo 3D ou sprite para representação visual.
Principais Classes e Estruturas ChessBoard: Classe responsável por representar o tabuleiro e gerenciar o posicionamento das peças.
Piece (e subclasses): Classe base para todas as peças de xadrez, contendo métodos comuns para movimentação e captura.
Player: Classe que representa um jogador no jogo, com métodos para realizar movimentos e interagir com o tabuleiro. GameManager: Controlador principal do jogo, responsável por gerenciar o estado do jogo e coordenar as interações entre os jogadores. Conclusão
Um jogo de xadrez desenvolvido com Unity, C# e POO oferece uma experiência interativa e educativa, permitindo aos jogadores explorar os conceitos do xadrez enquanto desfrutam de uma implementação moderna e visualmente atraente. A utilização da POO facilita a organização do código, tornando-o mais modular, extensível e fácil de dar manutenção. Além disso, a Unity oferece ferramentas poderosas para criar interfaces gráficas intuitivas e promover uma experiência imersiva para os jogadores.
